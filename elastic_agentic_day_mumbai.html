Agentic RAG: Grounding Agents in Enterprise Knowledge
Context rot is real. Giving an LLM more text makes it slower, less accurate, and more expensive. The answer isn't more data — it's better retrieval.

"More text is NOT the path to better AI. Accurate search is."

At 32K tokens, 11 models drop below 50% of their short-length baselines (NOLIMA, 2025)
H

Hybrid retrieval

BM25 + dense + ELSER sparse retrieval combined. Recall@10 of 84.3% — best of any single method.

C

Semantic chunking

93.3% hit rate. Reduces context window usage by 40%. Agents reason faster, cheaper, more accurately.

Q

Structured retrieval (ES|QL)

100% success rate on agentic tasks. Structured queries return exactly what the agent needs — nothing extraneous.

K

Knowledge tools

Runbooks, incident history, compliance docs, product specs — packaged as searchable tools agents call at runtime.

Session · 11:45 AM · Ashish Tiwari
MCP & A2A: Connecting Agents to Jira, Slack & Beyond
Agents are only as powerful as the tools they can reach. Open standards like MCP and A2A let your Elastic agents plug into — and be called by — any AI ecosystem.
MCP

Model Context Protocol

An open standard that lets AI models (Claude, Copilot, Cursor) call your Elastic agent as a tool. Generate an MCP key in Kibana — agents connect in under 5 minutes, no manual config.

A2A

Agent-to-Agent

Elastic agents expose A2A-compatible interfaces so orchestration frameworks like LangChain and AutoGen can delegate tasks directly — composing multi-agent pipelines with no lock-in.

What agents can do once connected

Session · 3:00 PM · Bhadriraju Prasanth
Governing AI Agents: Risk, Trust & Compliance
Autonomous agents acting at enterprise scale require more than good intentions. Governance is what separates a prototype from a production system your CISO will approve.
AC

Access control & RBAC

Elasticsearch's document- and field-level security ensures agents only retrieve data they're authorised to see. Role-based tool access limits what agents can act on.

AU

Full audit trail

Every retrieval, every tool call, every workflow execution logged to Elasticsearch. Immutable, queryable, and available for compliance reporting.

HL

Human-in-the-loop

Configurable approval steps in Elastic Workflows pause agent execution for high-risk actions. Trust is built incrementally — not assumed from day one.

GR

Guardrails & constraints

Agent Builder lets you define the exact tools, data sources, and actions each agent can use. Least-privilege by design — no open-ended autonomy.

PCI DSS 4.0.1 compliance assessments

Agent skills can run automated PCI compliance checks against your Elastic Security data — surfacing gaps, generating evidence, reducing audit prep time.

Data residency & multi-cloud

Elastic Cloud runs on AWS, Azure, and GCP across 50+ regions. Your data — and your agents — stay where your policies require.

Open Standards, No Lock-in: MCP, A2A & OpenTelemetry
The agentic ecosystem is maturing fast. Elastic bets on open standards — so your investments in automation, observability, and AI are portable, composable, and future-proof.
MCP

Model Context Protocol

The emerging open standard for tool-calling. Any MCP-compatible model (Claude, Copilot, Gemini CLI) can call your Elastic agents as first-class tools.

A2A

Agent-to-Agent

Google's A2A protocol enables multi-agent orchestration without vendor lock-in. Elastic agents are callable peers in any A2A-compliant pipeline.

OTel

OpenTelemetry

Elastic is a founding contributor to OTel. Your traces, metrics, and logs flow into Elasticsearch via open standards — no proprietary agents or SDKs required.
